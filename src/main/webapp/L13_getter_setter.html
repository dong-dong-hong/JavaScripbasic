객체지향 문법과 캡슐화와 getter setter(디자인 패턴)

  • Bean(완두콩) : 필드를 get set 캡슐화하는 모든 타입
  • VO : Bean인데 유효성 검사나 equals가 구현된 타입
  • DTO : Bean 데이터 전송에 사용하는 것
  • POJO : Plain Old Java Object의 줄임말 포조는 getter setter 디자인 패턴을 자바에서 처음 시작해서 부르는 말
      // Data Tranfer Object : 데이터 전송 시 보관용 캡슐화
    class PersonDto{
        private String name; // 은닉
        private int age;
        private boolean married;
        public setName(String name) {
          this.name = name;
        }
      public String getName() {
        return this.name;
      }
      public boolean isMarried() {
        return this.married;
      }
    }
      //VO Variable Object :  data를 저장하고 관리하는 용도 (잘 사용하지 않는다.)

자바스크리트의 캡슐화와 private protected (default는 없다 패키지 개념이 없어서)

    class Person{
        #password; // private : 클래스 내부에서만 접근 가능한 필드
        name; // public : 누구든 접근이 가능한 필드
        _child; // protected : 클래스 내부와 상속 관계에서 접근 가능한 필드
    }
